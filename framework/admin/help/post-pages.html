Post Pages

Post pages (or archive pages) are pages which display several posts, the most common of which is the blog page. Each of these pages has a separate section for complete control over their aspect. Also, every page besides the blog page can be set to inherit its options from the blog page.

  • Pagination - select the type of pagination to use on post pages. "Classic" has previous and next buttons, "Numbered" has previous and next arrows and numbered buttons, "Ajax On Click" has a button that will load posts via Ajax when clicked and "Ajax On Scroll" is the same as the on click one but loads posts when the bottom of the page is reached and the button comes into view.
Posts

This section contains options for the single post pages. Single posts can be styled differently than how they appear on archive pages. This allows for things like only displaying the post meta on the single post's page, and not also on the blog.

  • Post Tags - show a list of buttons for the tags the post has at the bottom of the post. Clicking on any tag will take the user to a page containing all posts tagged that specific term.
  • Post Navigation - enable this to show buttons to the previous and next posts after the content. This can be especially useful if you have a series of posts where each one continues from the last.
Post Meta

This section allows you to customize which meta information appears in the post meta element. Each component has a switch and an icon field (this can be left empty if you don't want icons to display. There is also a separator field if you want to use a special character to separate the meta components.

Post Formats

Here you can set the icon for each post format and type. Every field just takes the desired icon's class name(s), so no need to go into more detail.

Social Sharing

This is where you control the social aspects of your posts. There is a main switch to enable social sharing or not, and you can manage the profiles (or networks) that will show up in the sharing bar.

  • Shorten URLs - enabling this will use the Bit.ly service to shorten URLs used as the {link2} tag. This is useful for networks like Twitter, which limit the number of characters a message can contain. A username and API key will be needed.
  • Share URL Tags - to enable you to define any number of custom profiles, MIXT uses special tags in the "Share URL" field of the sharing profiles, which it then converts to data. A list of these tags is available below.
    • {site} - the site's name
    • {title} - the post's title
    • {link} - the post's permalink
    • {link2} - the post's shortlink or Bit.ly shortened URL
    • {thumb} - the post's thumbnail image
